ESP-32基于VS_CODE开发环境搭建

安装IDF环境离线包

去官网下载最新版本的IDF:

https://dl.espressif.cn/dl/esp-idf/

下载完成后,双击开始安装,安装第二页有一个”应用修复”,需要点击:

其他默认即可

VScode插件安装

首先在VScode安装idf插件:

安装完成后使用快捷键Ctrl+Shift+P,输入

接下来进入IDF-Setup界面

选择 “USE EXISTING SETUP选项” ，如果没有该选项,选择 “ADVANCED”

从Demo中新建工程

我们可以用SDK的Demo模板来创建新工程

使用快捷键Ctrl+Shift+P,输入

点击后会弹出 “Use current ESP-ID” 点击

左侧是Demo列表,右侧是功能描述,我们选择最简单的blink创建一个新工程

插件会以blink作模板,自动创建新工程,该工程功能是:配置IO为推挽输出,以1s间隔让灯亮灭切换,并用串口打印状态,模板中定义的BLINK_GPIO需要在menuconfig中定义,默认值是GPIO5

留意左下角的图标,从左到右分别是：选择下载端口、输出对象的硬件型号、当前工程、设置(插件化的 MenuConfig)、清理工程(clean)、编译(build)、烧录方式（UART/JTAG）、下载(flash)、监视窗口monitor（调试用）、一键 编译+下载+打开监视窗口、termainal终端。

点击第四个图标定义led管脚:

最后烧录、运行,成功后板子现象为LED以1s频率闪烁

留在最后

搭建过程中还是会存在一些问题的,不要气馁,可以联系我,希望能帮助你少走弯路
